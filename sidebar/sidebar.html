How to use scripts effectively

How it works: Uploaded PDFs are split into small text chunks and stored locally in your browser. When you ask a Q&A question, the extension finds chunks relevant to your current video position (±3 min) and your question, then includes those excerpts in the AI prompt.

Fuzzy vs. Semantic search:

  • Fuzzy matches based on character/word overlap. Fast and free, but struggles when your question uses different words than the script (e.g., asking about "convergence" won't find a section titled "limits").
  • Semantic uses a local AI model (all-MiniLM-L6-v2) to understand meaning. The extension bundles ~25 MB of ONNX/WebAssembly; the model file may download once (~25 MB) and is cached. It can match "convergence" to "limits" because it understands they're related. Inference runs 100% in your browser — nothing is sent to any server.

Best practices:

  • Seek to the relevant part of the video before asking — the system uses your current timestamp to find matching content.
  • Use Semantic search for best results; use Fuzzy for maximum speed and the smallest footprint (no embedding model).
  • Start with "Medium" script reliance and increase if answers lack script detail.
  • Upload only the script for the current course; unrelated PDFs dilute results.

Limitations:

  • Each retrieved chunk adds ~500 tokens. High reliance (14-20 chunks) can approach context limits on smaller models.
  • Heavily formatted PDFs (tables, diagrams, multi-column layouts) may extract poorly.
  • Semantic indexing takes 20-60 seconds per PDF on first upload (device-dependent). Subsequent queries are instant.
  • All processing is local — only matched excerpts are sent to the AI, never the full PDF.
